Per-user smart account. Owner (EOA) retains full authority;
session keys may call registered games within their scope.
Holds per-game balance slots.

### executeAsSession

Execute a call on behalf of a session key. Anyone may submit
this transaction (including a relayer) as long as the signature
and scope validate.


```solidity
function executeAsSession(
    address sessionKey,
    address game,
    uint256 value,
    bytes calldata data,
    uint256 nonce,
    bytes calldata signature
) external returns (bytes memory);
```
**Parameters**

|Name|Type|Description|
|----|----|-----------|
|`sessionKey`|`address`| The session key signer.|
|`game`|`address`|       Target game contract.|
|`value`|`uint256`|      Wei to forward from balances[game].|
|`data`|`bytes`|       Calldata (must include selector).|
|`nonce`|`uint256`|      Expected sessionNonce[sessionKey].|
|`signature`|`bytes`|  65-byte ECDSA signature over the digest below.|
