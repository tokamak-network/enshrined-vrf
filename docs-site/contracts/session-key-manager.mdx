Tracks scoped session keys for GameHub accounts.
Only the Hub itself may register/revoke keys for its own address.
Validation is pure view — callers (Hubs) enforce the result.
Designed to be a predeploy (address TBD).

## Functions
### register

Register a session key for the calling Hub.

msg.sender is the Hub. Only the Hub can register keys for itself.


```solidity
function register(address sessionKey, Scope calldata scope) external;
```

### revoke

Revoke a session key. Callable by the Hub that owns it.


```solidity
function revoke(address sessionKey) external;
```

### refill

Top up an active session key's remaining cap and/or extend its expiry.

Works on sessions whose expiry has passed (revives gameplay), but NOT on
revoked sessions — those must be re-registered from scratch.


```solidity
function refill(address sessionKey, uint256 addedCap, uint64 newExpiry) external;
```
**Parameters**

|Name|Type|Description|
|----|----|-----------|
|`sessionKey`|`address`| The key to refill.|
|`addedCap`|`uint256`|   Wei to add to `spendingCap`. May be zero if only extending expiry.|
|`newExpiry`|`uint64`|  New absolute unix timestamp. 0 leaves expiry unchanged; any non-zero value must be in the future.|


### canCall

Check whether a session key is authorized for a given call.
Pure view — does not mutate cap. Hub must call consume() after execution.


```solidity
function canCall(address hub, address sessionKey, address gameAddr, bytes4 selector, uint256 value)
    external
    view
    returns (bool);
```

### consume
