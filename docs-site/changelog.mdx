<Update label="2026-04-20" description="Phase 1 — Session Accounts">
  - Session-account contract spec and demo deployment: `GameHubFactory`, `SessionKeyManager`, and `GameRegistry`, with reserved fixed addresses for later enshrinement.
  - `refillSession` added to `GameHubAccount`; same-key top-ups in a single signature.
  - `GameHubFactory.depositAndRegister` / `depositAndRefill` for one-transaction onboarding and extension flows.
  - Example games: `DrawGame` (tier 0-99) and `CoinFlipGame` (heads/tails).
  - Foundry tests: 19 covering scope enforcement, refill semantics, cross-game isolation, and E2E VRF consumption.
</Update>

<Update label="2026-04-02" description="Architecture v2">
  - `PredeployedVRF` renamed to `EnshrinedVRF` across contracts and tooling.
  - Sequencer ECVRF commit path hardened; explicit seal key and attestation modes in `vrf-enclave`.
</Update>
