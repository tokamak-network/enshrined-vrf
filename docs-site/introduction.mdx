Enshrined VRF is a Layer 2 built around a simple conviction: **a chain for games should treat randomness and session-based authentication as first-class primitives**, not as third-party add-ons.

## Why this chain exists

Onchain games have two chronic friction points. First, **reliable randomness is hard**: app-level VRFs rely on oracle networks, which add latency and fees. Second, **wallet UX is hostile to play**: signing every in-game action destroys immersion.

This L2 collapses both problems into protocol features:

- **VRF** is a predeploy. Every game uses the same primitive. The sequencer commits once per block, and each call returns a unique value derived from that commitment.
- **Session accounts** are predeploys. A player deposits and registers a scoped session key in a single transaction, then plays indefinitely without signing, until the session hits its declared cap or expiry.

Everything else — games, frontends, tooling — builds on top.
