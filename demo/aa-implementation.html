어떻게 지갑 서명 없이
게임이 플레이되는가?

이 프로젝트는 EIP-4337을 쓰지 않습니다. EntryPoint도, Bundler도, UserOperation도 없습니다. 대신 3개의 컨트랙트와 브라우저에 저장된 임시 키 하나로 같은 UX를 만듭니다. 이 페이지는 그게 어떻게 가능한지 한 단계씩 해부합니다.

01

핵심 트릭 — "세션키"는 지갑의 키가 아닙니다

"서명 없이"가 가능한 이유는 단순합니다. 당신의 지갑 키는 서명하지 않지만, 브라우저가 만든 다른 키가 서명합니다. 이 다른 키를 세션키라고 부르고, 온체인에는 "이 세션키는 이 게임에만, N ETH까지만, T시까지만 허용"이라는 규칙이 기록됩니다.

EOA당신의 지갑 키

MetaMask / Rabby / Ledger
저장 위치지갑 앱 / 하드웨어
수명영구 (당신의 자산)
권한무제한 (모든 자산)
이 페이지에서 서명온보딩 1회만

단 한 번만 등장합니다. 게임 시작 전 "Hub에 입금 + 세션키 등록"이라는 한 트랜잭션에 서명하면 끝. 이후로는 지갑이 호출되지 않습니다.

이 페이지에서 서명플레이할 때마다 매번

브라우저가 페이지 로드 시 generatePrivateKey()로 즉석에서 생성한 새 키입니다. 당신의 지갑과 아무 관계가 없습니다. 이 키가 털려도 손해는 "남은 세션 한도" 뿐입니다.

02

등장 컨트랙트 — 이 4개가 전부입니다

EntryPoint도, Paymaster도, Bundler도 없습니다. 아래 4개의 역할을 이해하면 나머지는 조립 문제입니다.

PREDEPLOY

GameHubFactory

유저별 Hub을 CREATE2로 결정적 배포하는 공장. 주소는 owner만으로 계산 가능 — 즉, Hub이 아직 배포되지 않았어도 주소는 이미 알 수 있음.

  • hubOf(owner) → 주소 계산 (배포 전에도)
  • depositAndRegister(...) → 없으면 배포 + 입금 + 키 등록을 한 트랜잭션에
  • depositAndRefill(...) → 세션 한도 충전을 한 번에
PER USER

GameHubAccount (Hub)

당신의 미니 스마트 계정. 게임별 잔액 슬롯을 들고 있고, executeAsSession()에서 서명 검증 + 스코프 검증 + 실행을 전부 수행.

  • balances[game] → 게임별 earmarked 잔액
  • executeAsOwner(...) → EOA 직접 호출 경로
  • executeAsSession(sig, ...) → 세션키 경로 (지갑 불필요)
PREDEPLOY

SessionKeyManager

각 Hub이 등록한 세션키의 스코프 레지스트리. 키 ↔ (게임, 잔여 캡, 만료, 허용 셀렉터) 매핑. canCall()은 pure view — 강제력은 Hub에 있음.

  • register(sk, scope) → Hub만 호출 가능 (msg.sender 검증)
  • canCall(...) → 이 키가 지금 이 호출 허용? (view)
  • consume(amount) → 실행 후 캡 차감
ALLOWLIST

GameRegistry

허용된 게임 컨트랙트 목록. Hub은 미등록 게임에는 입금도, 호출도 거부. 이 한 겹이 있어서 세션키를 "임의 컨트랙트로의 백도어"로 쓸 수 없음.

  • isRegistered(game) → 화이트리스트 조회
  • 신규 게임은 거버넌스가 추가

05

세션키가 털렸을 때 — 피해가 제한되는 이유

세션키가 브라우저에 있다는 건 "상대적으로 안전하지 않은 키"라는 뜻이다. 그래서 4개의 게이트가 그 키의 권한을 사전에 좁힌다. 모두 SessionKeyManager.canCall() 한 함수 안에서 검증됨.

RAIL 1

게임 화이트리스트

이 세션키는 딱 이 게임 컨트랙트만 호출 가능. 다른 컨트랙트로는 호출 자체가 revert.

if (s.gameAddr != gameAddr) return false;
RAIL 2

스펜딩 캡

누적 사용액이 spendingCap을 넘으면 그 순간부터 거부. 매 실행마다 consume()으로 차감.

if (value > s.spendingCap) return false;
RAIL 3

만료 시간

등록 시 정한 unix timestamp를 지나면 키는 죽은 키. 유저가 탭을 닫은 뒤 긴 시간이 지나면 자동 무력화.

if (s.expiry <= block.timestamp) return false;
RAIL 4

수동 revoke

의심되면 EOA로 hub.revokeSession(sk) 한 번만 호출. 다음 블록부터 이 키는 영구히 무효.

if (!_active[hub][sk]) return false;

최악의 경우 손해: 현재 세션의 남은 cap. 지갑 내 다른 자산, 다른 게임 잔액, 다른 세션 — 전부 안전.

06

왜 EIP-4337을 쓰지 않았는가

4337도 결국 "계정이 자기 서명을 검증한다"가 핵심이다. 게임 도메인에서는 그걸 표준 스펙보다 훨씬 얇게 구현할 수 있다. Hub이 자기 자신의 EntryPoint 역할을 하므로, Bundler/EntryPoint/Paymaster 인프라가 필요 없다.

핵심 코드 보기 — executeAsSession() 
// GameHubAccount.sol:113
function executeAsSession(
  address sessionKey, address game, uint256 value,
  bytes calldata data, uint256 nonce, bytes calldata signature
) external returns (bytes memory) {
  // ① nonce 재사용 방지
  if (nonce != sessionNonce[sessionKey]) revert BadNonce();

  bytes32 digest = _digest(sessionKey, game, value, data, nonce);
  // ② 서명 ↔ 세션키 주소 매치 (Hub이 자기 자신의 EntryPoint 역할)
  address recovered = _recover(digest, signature);
  if (recovered != sessionKey) revert InvalidSignature();

  bytes4 selector = data.length >= 4 ? bytes4(data[:4]) : bytes4(0);
  // ③ SessionKeyManager에 스코프 질의 (게임 / 만료 / cap / 셀렉터)
  if (!sessionKeys.canCall(address(this), sessionKey, game, selector, value)) {
    revert Unauthorized();
  }
  // ④ 게임 잔액 확인
  if (balances[game] < value) revert InsufficientBalance();

  // 상태 갱신
  sessionNonce[sessionKey] = nonce + 1;
  balances[game] -= value;
  if (value > 0) sessionKeys.consume(sessionKey, value);

  emit SessionExecuted(sessionKey, game, selector, value);
  return _call(game, value, data);  // 게임 컨트랙트로 포워딩
}
핵심 코드 보기 — SessionKeyManager.canCall() (4개 레일) 
// SessionKeyManager.sol:82 — pure view, 변이 없음
function canCall(
  address hub, address sessionKey,
  address gameAddr, bytes4 selector, uint256 value
) external view returns (bool) {
  if (!_active[hub][sessionKey]) return false;          // RAIL 4: revoke
  Scope storage s = _scopes[hub][sessionKey];
  if (s.gameAddr != gameAddr) return false;              // RAIL 1: 게임 화이트리스트
  if (s.expiry != 0 && s.expiry <= block.timestamp)
    return false;                                        // RAIL 3: 만료
  if (value > s.spendingCap) return false;                // RAIL 2: 캡
  if (s.selectors.length > 0) {
    bool match_;
    for (uint256 i; i < s.selectors.length; ++i) {
      if (s.selectors[i] == selector) { match_ = true; break; }
    }
    if (!match_) return false;                            // 셀렉터 허용
  }
  return true;
}
