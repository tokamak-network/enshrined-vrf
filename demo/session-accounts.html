SESSION ACCOUNTS · DRAFT v0.1

한 번 서명하면, 게임 내내 무서명.

enshrined-vrf L2에 세션 계정을 내장해, 유저가 입금할 때만 서명하고 게임을 플레이하는 동안에는 팝업 없이 진행하도록 합니다. VRF 기반 게임의 몰입감을 지키는 것이 목표입니다.

01문제와 해결

기존 지갑은 매 액션마다 서명을 요구합니다. 게임에선 치명적입니다.

현재: 매 액션마다 서명

🔐 sign → move()
🔐 sign → attack()
🔐 sign → drawRandom()
🔐 sign → bet()
🔐 sign → move() ...

팝업이 게임 흐름을 끊고, 유저는 곧 피로해집니다. 온체인 게임이 성장하지 못하는 주요 이유.

Session Accounts: 입금 시 1회

🔐 sign → deposit + register session
· move()
· attack()
· drawRandom()
· bet() ... (모두 서명 없음)

세션키가 정해진 게임·한도·기간 안에서 유저를 대신해 서명합니다. 유저는 그 존재를 의식하지 않습니다.

023계층 계정 모델

메인 EOA는 자산을 지키고, Hub는 게임 잔고를 관리하며, 세션키는 게임별 스코프 안에서만 작동합니다.

LAYER 1 · FULL CUSTODY
EOA (메인 지갑)
유저의 실제 지갑. 큰 잔고 보관. 서명 권한 풀셋.
│
▼ deposit + register (서명 1회)
LAYER 2 · PER-USER SMART ACCOUNT
GameHub Account
유저당 1개. 게임별 잔고 슬롯. 세션키 서명 검증. CREATE2 결정적 주소.
SessionKey_A
→ Game A 컨트랙트
cap 1 ETH · 24h
SessionKey_B
→ Game B 컨트랙트
cap 0.5 ETH · 6h
SessionKey_C
→ Game C 컨트랙트
cap 2 ETH · 12h
│
▼ session key signs (유저 서명 없음)
LAYER 3 · GAME CONTRACTS
Registered Games (Game A, B, C, ...)
GameRegistry 화이트리스트에 등록된 컨트랙트만 세션키로 호출 가능. 모두 EnshrainedVRF 공유.

03세 가지 트랜잭션 흐름

서명이 필요한 순간은 입금과 출금뿐. 플레이는 무서명.

FLOW · DEPOSIT

입금 + 세션 등록

유저가 처음으로 게임을 시작할 때의 단일 트랜잭션.

1EOA가 GameHubFactory.depositAndRegister 호출
2Hub 미존재 시 CREATE2로 배포
3Hub의 balances[gameAddr] += X
4SessionKeyManager에 (hub, key, scope) 등록
🔐 signsEOA × 1
FLOW · PLAY

게임 플레이

매 턴마다 실행. 유저에겐 서명 팝업이 뜨지 않음.

1클라이언트가 세션키로 tx 서명
2Hub의 validateTransaction 실행
3SessionKeyManager가 scope · cap · expiry 검증
4GameRegistry로 target 화이트리스트 확인
5게임 컨트랙트 호출 → EnshrainedVRF 등 사용
✓ no popupsession × ∞
FLOW · WITHDRAW

출금

자금을 EOA로 되돌릴 때만 메인키 서명 필요.

1EOA가 Hub.withdraw 호출
2Hub: require(msg.sender == owner)
3세션키는 출금 권한 없음
4(선택) 챌린지 기간 후 정산
🔐 signsEOA × 1

05확정된 설계

이번 문서에서 고정. 변경 시 문서를 먼저 수정.

✓

3계층 계정 모델

EOA → GameHub → SessionKeys.

✓

Hub는 유저당 1개

게임당이 아닌 유저당. 잔고는 Hub 내부 슬롯으로 격리.

✓

GameRegistry 게이팅

등록된 게임 컨트랙트만 세션키로 호출 가능.

✓

세션키는 출금 불가

출금은 오직 메인 EOA 서명으로만.

✓

컨트랙트 우선 구현

Solidity 프로토타입으로 semantics 확정 후 op-geth로 enshrine.

✓

VRF 재사용

별도 추가 없이 기존 EnshrainedVRF를 게임이 공유.
