01 / THE PROBLEM

Sequencer knows sk →
unpredictability is broken

VRF is verifiable — anyone can check the proof. But if the sequencer holds sk, it can compute VRF(sk, future_seed) before building the block, breaking unpredictability. The key flows through 4 files in plain Go memory.

Unpredictability Broken

sk + seed → the sequencer can compute VRF output before the block exists. It knows every coin flip, dice roll, and lottery result in advance.

2

Tx Ordering Manipulation

With predicted randomness, the sequencer can reorder, insert, or exclude transactions to extract value from users.

=

Verifiable ≠ Unpredictable

VRF guarantees verifiability (anyone can check proofs). But it does NOT guarantee unpredictability if the prover is the sequencer itself.

02 / THE SOLUTION

Hide sk in TEE →
unpredictability restored

The sequencer must not know sk. A TEE enclave holds the key in hardware isolation — the sequencer sends a seed and receives the proof, but cannot access sk. This restores unpredictability while keeping verifiability and zero-latency UX.

03 / TEE ENCLAVE

Secret key lives only inside
the hardware enclave

The enclave server generates the VRF key on first run, seals it to encrypted storage, and serves proofs over gRPC. The key never leaves — not even during restarts.

type Server struct { pb.UnimplementedVRFEnclaveServer sk *secp256k1.PrivateKey // only here, nowhere else pk []byte } func NewServer(storage *SealedStorage) (*Server, error) { if storage.Exists() { skBytes, _ := storage.Unseal() // decrypt from disk sk = secp256k1.PrivKeyFromBytes(skBytes) } else { sk, _ = generateKey() // fresh random key storage.Seal(sk.Key.Bytes()[:]) // encrypt to disk } return &Server{sk: sk, pk: sk.PubKey().SerializeCompressed()} } func (s *Server) Prove(ctx, req) (*ProveResponse, error) { beta, pi, _ := ecvrf.Prove(s.sk, req.Seed) // sign inside enclave return &ProveResponse{Beta: beta[:], Pi: pi[:]} }

04 / ON-CHAIN

Contracts and precompile
make it all transparent

Three on-chain components work together: the predeploy stores committed randomness, the precompile verifies proofs, and user contracts consume randomness synchronously.

→

0x42..00f0

P

PredeployedVRF @ 0x42..f0

Stores VRF results. Only DEPOSITOR can commit. Anyone can consume via getRandomness().

V

Verify Precompile @ 0x0101

On-chain ECVRF verification. staticcall with (pk, seed, beta, pi) returns 0x01 if valid. 3,000 gas.

C

Consumer Contracts

Call VRF.getRandomness() like any other function. Same block, zero delay, fully synchronous.

PredeployedVRF.sol 0x42..f0
address constant DEPOSITOR = 0xDeaD..0001; /// Sequencer commits VRF result as first tx in block function commitRandomness( uint256 nonce, bytes32 seed, bytes32 beta, bytes pi ) external onlyDepositor { _results[nonce] = VrfResult(seed, beta, pi, block.number); emit RandomnessCommitted(nonce, beta); _commitNonce++; } /// User contracts call this — returns unique randomness per call function getRandomness() external returns (uint256) { return uint256(keccak256(abi.encodePacked(_currentBeta, _callCounter++))); }
CoinFlip.sol CONSUMER
contract CoinFlip { IEnshrainedVRF constant VRF = IEnshrainedVRF(0x42000000000000000000000000000000000000f0); function flip() external returns (bool heads) { uint256 randomness = VRF.getRandomness(); heads = (randomness % 2 == 0); } }
ECVRF Verify PRECOMPILE 0x0101
/// Anyone can verify a VRF proof on-chain for 3,000 gas /// /// Input: abi.encodePacked(pk, seed, beta, pi) /// Output: 0x01 = valid, 0x00 = invalid function verifyOnChain(bytes pk, bytes32 seed, bytes32 beta, bytes pi) view returns (bool) { (bool ok, bytes memory out) = address(0x0101).staticcall( abi.encodePacked(pk, seed, beta, pi) ); return ok && out[0] == 0x01; }
